# Scope Context Guardrails

When editing the seeded example or example-specific docs in this repo:

- preserve the stated example intent unless the user explicitly asks to change
  it
- keep example-local assumptions inside the example surface, not repo-wide
  canon
- confirm whether the issue is a bug, tooling problem, or misunderstanding
  before changing behaviour
- collect evidence before broadening scope

The example exists to exercise infrastructure, not to define the whole repo.
