# PDR-024: Vital Integration Surfaces

- Status: Accepted
- Date: 2026-04-23

## Context

A repo can look Practice-shaped while still being inert. A few markdown files
are not enough; the Core must connect to live repo surfaces in both
directions.

## Decision

The Practice is considered operational only when it has, in some truthful host
form:

- an entry-point chain into `AGENT.md`
- a local bridge in `.agent/practice-index.md`
- session-start skills
- active memory surfaces
- continuity surfaces
- graduation workflow
- validators or audits that protect the contracts

## Consequences

- verification must check for live surfaces, not only copied files
- missing integration surfaces are structural defects
- a fresh checkout should still expose the same governed estate
