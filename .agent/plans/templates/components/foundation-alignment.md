# Component: Foundation Alignment

At plan start and at the start of each phase, re-read:

1. `.agent/directives/principles.md`
2. `.agent/directives/testing-strategy.md`
3. `.agent/directives/orientation.md`
4. `.agent/directives/data-boundary-doctrine.md`

## Per-phase questions

1. Does this move the repo toward the stated end goal?
2. Could it be simpler without compromising quality?
3. Are we proving the right behaviour in the right layer?
4. Are boundaries explicit and validation happening at the boundary?

## End-of-plan checklist

- [ ] no compatibility layers were introduced
- [ ] no quality gates were weakened or bypassed
- [ ] validation stayed at the boundary
- [ ] tests prove behaviour, not incidental implementation
- [ ] durable docs were updated or explicitly ruled unchanged
