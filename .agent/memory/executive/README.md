# Executive Memory

Executive memory holds stable catalogues and contracts about how this repo's
agentic infrastructure is organised.

These files are looked up when a governed action is happening. They are not
part of the per-session capture loop.

## Surfaces

| Surface | Purpose |
| --- | --- |
| [`artefact-inventory.md`](artefact-inventory.md) | Canonical-vs-adapter inventory for agent artefacts |
| [`invoke-code-reviewers.md`](invoke-code-reviewers.md) | Reviewer catalogue and dispatch guidance |
| [`cross-platform-agent-surface-matrix.md`](cross-platform-agent-surface-matrix.md) | Supported and unsupported platform/surface pairs |

## Refresh Discipline

Amend executive memory only when the artefact architecture itself changes:

- a new governed surface is introduced,
- a reviewer roster changes,
- a platform support claim changes,
- or the canonical-vs-adapter contract is revised.
