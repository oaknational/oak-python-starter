# Consolidate Docs

This is the slower closeout workflow. It is thread-scoped rather than
session-scoped.

Use it when:

- incoming Practice material needs integration
- plans or memory are carrying durable substance
- the documentation tiers need routing or promotion
- continuity or executive surfaces have accumulated drift

## Cardinal Rule

Completed plans must be safe to archive. If a plan contains durable guidance,
home that guidance before the plan is closed.

## Steps

1. **Run the homing method.** Use
   [`ephemeral-to-permanent-homing`](ephemeral-to-permanent-homing.md) to move
   durable content out of plans, memory, and entry surfaces.
2. **Refresh plan surfaces.** Keep `high-level-plan.md`, collection indexes,
   and `completed-plans.md` truthful.
3. **Graduate memory deliberately.** Promote settled lessons into directives,
   rules, ADRs, PDRs, READMEs, reports, or reference material.
4. **Audit the document tiers.** Route exploratory material to `research/`,
   investigations to `analysis/`, formal syntheses to `reports/`, evergreen
   library content to `reference/`, and durable option-weighing to
   `docs/explorations/`.
5. **Audit experience.** Keep experience files subjective; rescue any durable
   technical insight to the proper permanent home.
6. **Refresh indexes and bridges.** Keep the Practice Index, collection READMEs,
   and reference indexes aligned with reality.
7. **Rotate the napkin when needed.** Distil what matters, archive when useful,
   and keep `distilled.md` lean.
8. **Check fitness.** Tighten overgrown directives, PDR indexes, plan indexes,
   and tier READMEs.
9. **Integrate incoming Practice.** Compare incoming material concept-by-concept,
   preserve stronger local contracts, land the useful changes, and clear the
   incoming box.
10. **Keep adapters thin.** Update wrapper surfaces and parity checks in the
    same landing as canonical changes.

## Outputs

By the end of this workflow:

- plans no longer carry durable doctrine
- the memory, analysis, reports, reference, and experience tiers are routed
  truthfully
- incoming Practice material is integrated or explicitly rejected
- live docs and adapters agree with the filesystem
